# PCワークアセスメント（仮称）

就労選択支援におけるアセスメントで使用する、PC事務作業の評価・訓練ツールです。
ワークサンプル幕張版（MWS）のOA作業の**思想（課題分析に基づく難易度段階・簡易版/訓練版の2モード構成）を参考にした独自設計**であり、MWSの問題・画面・データは一切使用していません。

## 設計上の不変条件

- **個人情報は一切保存しない。** 利用者は任意のID（例：A-01）のみで識別する。氏名・生年月日などの入力欄は作らない。
- **完全ローカル動作。** サーバー・ネットワーク不要。データはブラウザのlocalStorageのみに保存。
- **機械的な判定で本人の評価を決めつけない。** 結果表示は「〜の可能性があります」のトーンで統一し、支援者の観察と合わせて判断する旨を常に併記する。
- **失敗体験への配慮。** 適応モードでレベルが下がっても本人に通知しない。結果画面は「できたこと」を先に表示。効果音・派手なアニメーションなし。正誤は色だけでなく記号（○/×）でも表示。

## 起動方法

### 公開URL（推奨・インストール不要）

**https://heatonmeromero.github.io/pc-work-assessment/**

ブラウザ（Edge / Chrome）で開くだけで使えます。アプリはサーバーと通信せず、
記録データは**開いたPCのそのブラウザの中**にだけ保存されます（URLを知っている他の人が
開いても、その人のブラウザに新しい空のデータができるだけで、こちらの記録は見えません）。

### 配布版（オフラインのPC向け）
1. `dist` フォルダごとUSB等でコピーする
2. `dist` 内の `start.bat` をダブルクリック（または `index.html` をブラウザで開く）

ビルドは単一HTMLに全アセットを内包しているため、`file://` のまま動作します。Edge / Chrome（Windows 11）で動作確認。

### 開発・公開版の更新
```
npm install
npm run dev        # 開発サーバー
npm run build      # dist/ に単一HTMLを出力（配布版）
npm run build:web  # docs/ に出力（公開URL用）
git add -A && git commit -m "更新内容" && git push   # → 公開URLに自動反映（1〜2分）
```

## データの保存と移動（重要）

- データは **「そのPCの・そのブラウザ」のlocalStorage** にのみ保存されます。EdgeとChromeでデータは共有されません。**運用するブラウザを1つに決めてください。**
- バックアップ・PC間移動は、管理画面の **「全データをJSONエクスポート」→ 別PCで「JSONをインポート（マージ）」** で行います。
- CSVエクスポート（問題ごとのローデータ）はExcel集計・支援記録への転記用です（BOM付きUTF-8）。
- ブラウザの「閲覧データの削除」を行うと記録が消えます。定期的なJSONバックアップを推奨します。
- セッション実施中にブラウザが落ちた場合も、1問ごとに逐次保存しているため、次回起動時に「中断データ」として復元されます。

## 運用保護機能（v0.8）

本番前リスク点検の結果を受けて追加した保護機能です。

- **管理画面PIN**：支援員共通の合言葉（4〜8桁）。初回に設定し、以後は入室時と「全データを削除」実行時に必要。
  PINを忘れても「PINを忘れた場合」から初期化でき、**記録データは消えません**
  （※誤操作・いたずら防止が目的の仕組みで、暗号化はしていません）。
- **複数タブのブロック**：同じブラウザで2つ目のタブ/ウィンドウを開くと操作をブロック（記録の上書き消失防止）。
- **保存失敗の通知**：容量不足などで保存できなかったとき、赤いバナーで知らせます。管理画面に容量メーターあり。
- **CSVの安全化**：Excelの数式実行防止／日時は日本時間表記／数字だけのIDは文字列として保持。
- **ID・メモの運用ガード**：初回起動時に運用注意を表示。氏名らしきIDには確認を表示。
- **未保存メモの警告**：所見メモを保存せずに画面を移動しようとすると確認します。

ロールバック（公開版を前の状態に戻す）：`git revert HEAD && git push` で1〜2分後に反映。

### 総合レポート

モード選択の「総合レポート」から、利用者IDの全セッションを集計して表示します（A4縦で印刷可）。
1. **5種目プロフィール**：種目別めやすレベルのレーダーチャート＋根拠つき一覧表
   （適応モードの推定値を優先。なければ正答率70%以上・5問以上で取り組めた固定レベルの実績）
2. **速度と正確性**：散布図。横軸は「その種目・レベルの標準時間との比」なので種目をまたいで比較可能
3. **持続性**：全セッションの前半/後半集計（6問以上のセッションが対象）
4. **学習曲線**：同種目・同レベルを3回以上くり返した場合の推移（正答率と平均時間）
5. **エラー傾向と配慮の参考例**：頻出エラー上位と参考例文。すべて「〜の可能性があります」の
   断定しないトーンで、支援者の観察と合わせて判断する前提を明記
6. **自己評価と実測のくらべ**：本人の自己評価とのずれを自己理解の材料として提示
7. **所見メモ**：支援者の自由記述（エクスポート・印刷に含む）

印刷はブラウザの印刷機能（レポート画面の「印刷（A4）」ボタン）。操作ボタン類は印刷に含まれません。

### アセスメント（総合）モード

5種目を順番に実施します（数値入力 → 文書入力 → コピー＆ペースト → 検索修正 → ファイル整理）。
- **合計時間**（10/15/20分/カスタム）を、種目×レベル別の標準時間テーブルに比例して自動配分
  （どの種目もおおむね同じ問題数に取り組める設計）。配分は設定画面で確認できます。
- **レベルの決めかた**：自動調整（おすすめ・種目ごとのめやすレベルを測定）または固定レベル。
- 種目のあいだは説明画面で止まるため、そこで休憩できます。各種目の練習問題も設定可能。
- 自己評価は最後に全体をとおして1回。1種目終わるごとに保存されるため、途中でブラウザが
  落ちてもそれまでの種目の記録は残ります。
- 通し結果画面：種目別の正答率・平均時間・レベルのめやす比較、種目別詳細へのリンク、
  所見メモ、JSON/CSVエクスポート。

### ファイル整理の記録項目

疑似ファイルマネージャー上で、ドラッグ＆ドロップ（またはクリック選択→フォルダクリック）で分類します。
実OSのファイルには一切触れません。
- **分類正答率**と、まちがいの**条件別の内訳**：種類（拡張子）／年（日付）／例外条件
  「ただし〜」の見落とし／不要ファイルの見分け／未分類
- **迷い**：同じ場所へ戻した・ドラッグを途中でやめた回数
- **置きなおし**：一度置いたファイルを動かしなおした回数

レベル進行：種類で分ける（L1〜3）→ 年で分ける（L4）→ ゴミ箱混在（L5）→
2階層・年→種類（L6〜7）→ 例外条件「請求を含むPDFは経理へ」（L8）→ 全要素＋40ファイル（L10）。
※ 仕様にあった「リネーム」混在は判定モデルが複雑になるため未実装（必要なら追加可）。

### コピー＆ペーストの記録項目

正確性（取り違え／選択範囲の過不足／未入力／内容相違）に加えて、操作ログを記録します。
- **選択 → コピー → 貼り付け** の各回数（段取りの効率の材料）
- **ショートカット利用率**：Ctrl+C/V か右クリックメニュー経由かを判別（操作習熟の材料）
- **貼り間違い・やり直し**：期待値と異なる内容を貼り付けた回数
- **手入力キー数**：コピペを使わず手で打った量（方略の観察材料）

レベル進行：リストから1項目（L1）→ 表から該当セルを探す（L4〜）→
貼り付け先の順序が元と異なる並べ替え（L6〜）→ 長い備考文・似たコードの大きな表（L8〜10）。

### 検索修正の記録項目

セルごとに「原本／提示値（誤りを含む）／利用者の確定値」を突き合わせて判定します。
- **発見・修正率**：埋め込んだ誤りのうち正しく直せた割合
- **見逃し**：手をつけなかった誤り、**直したが不正確**：触ったが原本と一致しない
- **誤検出**：正しい箇所を変えてしまった数
- **見落とし位置**：見逃した誤りの左右（左/中央/右）・上下（上/中/下）の位置をローデータに記録（半側空間無視などの気づきの材料。診断はしない）

誤りの種類は、数字のちがい・数字の入れかわり・かな/カタカナの混同・似た漢字・全角/半角の5種を
レベルに応じて巧妙化させます（高レベルほど密度が下がり、探す負荷が上がる）。

### 文書入力の「変換誤り」検出について

問題文は語彙プール（各語に読みを付与）から合成されるため、見本と入力の両方を
同じ手続きで「読み形」に変換し、**表記は違うが読みが一致する相違を「変換誤り」**
として分類します（例：精算→清算、回答→かいとう（未変換））。
辞書にない同音異義語（変換候補に出ない造語など）は「別の文字の入力」として
数えられる限界があります。判定はあくまで参考情報です。

## 構成

```
src/
  engine/      乱数(シード付き)・ステアケース法・文字diff
  store/       localStorage・エクスポート/インポート・アプリ状態
  tasks/       種目プラグイン（taskModule.ts のインターフェースに従って追加）
    numeric/   数値入力（生成・採点・UI）
  screens/     トップ/モード選択/設定/セッション実行/結果/履歴/管理
  components/  SVGチャート（依存ライブラリなし）
```

新しい種目は `src/tasks/<種目>/` を作り、`TaskModule` インターフェースを実装して `registry.ts` に登録すると、設定画面・セッション実行・結果画面・CSVに自動的に組み込まれます。
